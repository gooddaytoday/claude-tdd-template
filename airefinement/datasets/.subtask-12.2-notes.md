## Dataset Schema per Task

- `id` — unique string identifier
- `type` — `"unit"` | `"integration"` | `"both"`
- `difficulty` — `"easy"` | `"medium"` | `"hard"`
- `reference_solution` — `""` (to be filled after real eval runs)
- `graders` — `["deterministic"]`

## Integration Test Coverage

Test file `tests/integration/eval/dataset-validation.test.ts` validates:
1. File `datasets/golden-v1.jsonl` exists
2. Contains >= 5 tasks
3. Every task conforms to the schema (required fields, valid enum values)
4. All task IDs are unique
5. Both `unit` and `integration` types are present in the dataset

## Architectural Notes

- JSONL format (one JSON object per line) chosen for easy streaming and append-only extension
- `reference_solution` is intentionally empty; will be populated after first batch of real eval runs (subtask 12.3+)
- `graders` defaults to `["deterministic"]`; future tasks may add `"llm-judge"`
